# Explanation Standard

Use this standard before writing or revising question explanations.

## Requirements

- Verify first using source context and reliable references.
- Prefer textbook/source context over broad web results.
- Use reliable external references when the answer depends on factual knowledge outside the source.
- Keep an evidence note in the workspace or release metadata when external references are used.
- The app-facing explanation must be accurate, plain-language, and short: at most 2 display lines.
- Do not change stems, options, answer keys, or answer text to make an explanation fit.
- If the explanation cannot be verified, create a blocking ticket.

## Style

Default format:

```text
本题考查……。正确答案是……，因为……。
```

Good explanations:

- state the tested point directly
- mention the key reason for the correct answer
- mention a common confusion only when useful
- avoid long background summaries

If 2 lines are needed, the second line should be a confusion hint or memory hint, not an essay.

## Forbidden

- Do not invent facts.
- Do not rely on a likely answer without verification.
- Do not write vague explanations such as `见课文内容` or `常识题`.
- Do not copy large external passages into explanation text.
- Do not expose source URLs in the app-facing explanation unless the question itself asks about a source.
